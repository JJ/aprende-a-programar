Pero se sigue usando Java y C++ para enseñar, o mejor, para tratar de enseñar, cursos introductorios a la programación en las universidades; pero no sólo en las universidades, también en formación de empresa, ciclos y, en general, en casi cualquier lugar que se enseñe programación.

Si los filósofos fueran programadores, Wittgenstein sería Haskell. Witgenstein habla del lenguaje como algo que no puede separarse de las acciones que lleva a cabo. A través de sus "juegos del lenguaje", analiza la imbricación de lenguajes y acciones creando mini-lenguas que sirven en un entorno determinado, que les da significado. La interpretación posmoderna es también que el medio es el mensaje, por lo que el enseñar un lenguaje de programación de una forma determinada transmite un mensaje. El problema es que, los juegos de lenguaje son como las bromas privadas. Sólo puedes entenderlo si estás en el mismo contexto. Trabajar con lenguajes en un contexto de hace 20 años hace mucho más complicado entenderlos y, por supuesto, aprender a hacer tareas de forma eficiente usando ese o, para el caso, cualquier otro lenguaje.

Los lenguajes no están separados de la realidad

Es una de las afirmaciones de sus investigaciones filosóficas. Los lenguajes no describen una realidad, sino que la definen y la restringen. Un lenguaje que te permita hacer sólo un tipo determinado de cosas, te restringe lo que puedas hacer en el futuro. Si escribes a consola, estás aprendiendo a tener una realidad en la que se escribe a consola. En sus "juegos del lenguaje" trata de poner de manifiesto que un lenguaje es parte de una forma de vida, a la que le da significado.
Entre otras cosas, lo importante es ver que para aprender a programar no hace falta completar un programa. La mayoría de los lenguajes interpretados, y algunos compilados, tienen un REPL desde el que se pueden evaluar expresiones, guardar en variables resultados y hacer un montón de cosas. package main
import (
	"fmt"
	"net/http"
	"os"
)
func main() {
	http.HandleFunc("/",
		func (res http.ResponseWriter, req *http.Request) {
			fmt.Fprintln( res, `<html>...</body></html>`)
		})
	ε := http.ListenAndServe(":"+os.Getenv("PORT"), nil)
	if ε != nil { panic( ε ) }
}
Trabajar con variables de entorno es una buena práctica que se usa, por ejemplo, para despliegues en la nube de forma segura. Permite, además, definir las entradas una sola vez, sin tener que repetirlas cada vez que se ejecute. Es un ejemplo de cosas que no se suelen cubrir con el "Hola mundo": trabajar con el entorno y que, sin embargo, resultan imprescindibles
